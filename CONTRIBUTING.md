# Contributing to DeepAgents

Thank you for your interest in contributing! This document provides guidelines for contributing to this project.

## How to Contribute

1. **Fork the Repository**: Create your own fork of the project
2. **Create a Branch**: Create a feature branch (git checkout -b feature/amazing-feature)
3. **Make Changes**: Implement your changes with clear, focused commits
4. **Test**: Ensure your changes don't break existing functionality
5. **Submit a Pull Request**: Open a PR with a clear description of your changes

## Code Style

- Follow existing code formatting and style
- Write clear, self-documenting code
- Add comments where necessary to explain complex logic
- Keep functions focused and modular

## Pull Request Process

1. Update documentation if your changes affect usage
2. Ensure your code follows the project's style guidelines
3. Include relevant tests if applicable
4. Link any related issues in your PR description
5. Be responsive to feedback and requested changes

## Reporting Bugs

When reporting bugs, please include:
- Clear description of the issue
- Steps to reproduce
- Expected vs actual behavior
- Environment details (OS, version, etc.)

## Feature Requests

We welcome feature suggestions! Please:
- Check existing issues first
- Describe the feature and its use case
- Explain why it would be valuable

## Questions?

Feel free to open an issue for questions or join discussions.

## Code of Conduct

By participating, you agree to maintain a welcoming and respectful environment for all contributors.
